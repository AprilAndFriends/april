enum april::Texture::AddressMode

Enumerator
ADDRESS_WRAP 

Wrapping UV coordinates.

ADDRESS_CLAMP

enum april::Texture::Filter

Defines texture filtering modes.

Enumerator
FILTER_NEAREST 

Nearest neighbor.

FILTER_LINEAR 

Linear interpolation.

enum april::Texture::LoadMode

Defines how and when textures should be loaded into VRAM.

Enumerator
LOAD_IMMEDIATE 

Loads the texture and uploads data to the GPU right away.

LOAD_ON_DEMAND 

Doesn't load the texture yet at all. It will be loaded and uploaded to the GPU on the first use.

LOAD_ASYNC 

Loads the texture asynchronously right away and uploads the data to the GPU as soon as it is available before the next frame.

Note
If the texture is used before it loaded asynchronously, it cannot be uploaded to the GPU and will not be rendered properly.
LOAD_ASYNC_ON_DEMAND 

Loads the texture asynchronously right away, but it will upload the data to the GPU on the first use.

Note
If the texture is used before it loaded asynchronously, it cannot be uploaded to the GPU and will not be rendered properly.
enum april::Texture::Type

Defines texture types in order to control their behavior and certain features.

Enumerator
TYPE_MANAGED 

Resides in RAM and on GPU, can be modified. Best used for manually created textures or loaded from files which will be modified.

TYPE_IMMUTABLE 

Cannot be modified or read. Texture with manual data will have a copy of the data in RAM, files will be reloaded from persistent memory.

TYPE_VOLATILE 

Used for feeding the GPU texture data constantly (e.g. video). It has no local RAM copy for when the rendering context is lost and cannot be restored.

TYPE_RENDER_TARGET 

Used for render targets. Acts like MANAGED.
